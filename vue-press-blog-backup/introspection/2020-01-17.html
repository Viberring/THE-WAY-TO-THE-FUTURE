# Learning Programming Language

Link: inspire

# 语法和语义

程序始终充满各种抽象(控制流，函数，类...)
各种语言有不同的语法，但是始终是万变不离其宗。
所以在学习新语言的时候，应该关注的特定语言对于那些抽象的具体实现。 这些实现在改语言中代表的特定意义，就好比有一个scope，在编程中脱离scope而去谈论东西就像是天方夜谭。
所以我们更应该去了解语言的特点，与其他语言的特别之处（如数据结构等的不同实现）。
上面的前提是，你已经对某种语言很熟悉（比如精通c++ >_<~~)。

如Lisp，它早已不是单纯的一种语言，而是一种语言系，对于此类的语言，应该去了解这些针对于lisp不同实现的异同，设计思想的不同，从而更进一布的全方面去了解。 比如，现在我就应该去更多的了解c语言系。

# 概念

计算机的概念太多了。
目前在编程语言中的概念，普遍的是这些
函数，类，对象，封装，多态，类型推导，结构体，指针 等。
应该熟练的掌握这些抽象的概念。

# 实现

如果能自己实现一门语言，那么学习一门新的语言，你就知道如何去理解。
比如，为什么它要这样实现，以及这样实现的好处。
我应该去试着编写解释器和编译器。 因为如果你去完成类这些事情，你就懂了。

# 思维

语言是用来表达思维的，我们应该享受将自己的思维付诸于代码来实现的这个过程，程序是思维的延续， 而程序能够运行无误则是验证思维是否有偏差的证据。 毕竟这是一门实践的艺术，所以要去实践，实践，实践。 我发现，我应该提升自己独立思考，独立实践的能力，你需要看很多的代码，但是看很多的代码不能让你成为厉害的人，应该是写很多的代码，所以，我应该去实践，去写程序，我现在的实践太少。

# 心态

没有任何语言是完美的
语言有它的优点和缺点，应该去学习语言的优点，而不是去抨击语言的缺点。
对于自己，我应该明确自己的缺点，然后去弥补。更应该，胸襟宽广，海纳百川。
