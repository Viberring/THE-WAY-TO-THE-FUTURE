# What is Aop

Aspect Oriented Programming (AOP)

AOP complements OOP by providing another way of thinking about program structure.
The key unit of modularity in OOP is the class, whereas in AOP the unit of modularity is the aspect.
Aspect enable the modularization of concers such as transaction management that cut across multiple types and objects.

  • Annotation

@Retention(RetentionPolicy.RUNTIME)

public @interface LogAnnotation {

}
  • Aop
@Aspect
@Component
@Order(100)
public class LogAspect {

    @Pointcut("execution(* com.aop.controller..*.*(..))")
    public void log() {

    }

    @Before(value = "@annotation(la)")
    public void doBefore(JoinPoint joinPoint, LogAnnotation la) {
        long startTime = System.currentTimeMillis();
        System.out.println(startTime);
        System.out.println(joinPoint.getArgs());
        System.out.println("before........");
    }

    @AfterReturning(value = "@annotation(la)")
    public void doAfter(JoinPoint joinPoint, LogAnnotation la) {
        System.out.println("returning.....");
    }

    @AfterThrowing(value = "@annotation(la)")
    public void doThrow(JoinPoint joinPoint, LogAnnotation la) {
        System.out.println("throwing.......");
    }

    @Around(value = "@annotation(la)")
    public Object aroundDo(ProceedingJoinPoint joinPoint, LogAnnotation la) throws Throwable {
        System.out.println();
        try {
            return  joinPoint.proceed();
        } catch (Throwable throwable) {
            return throwable;
        }
    }
}
